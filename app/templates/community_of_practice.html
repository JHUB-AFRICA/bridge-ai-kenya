📋 Contribution Guidelines

How to Contribute

Everything you need to know to start contributing to BRIDGE-AI projects

Browse our GitHub repositories and look for issues labeled "good first issue" or "help wanted". These are great starting points for new contributors.

  • • Check the issues tab on each repository
  • • Look for labels like "good first issue"
  • • Comment on the issue to express interest

Create your own copy of the repository by forking it on GitHub. This gives you a personal workspace to make changes without affecting the main project.

  • • Click the "Fork" button on the repository page
  • • Clone your fork locally
  • • Create a new branch for your work

Implement your changes following the project's coding standards and best practices. Write clear, well-documented code.

  • • Follow PEP8 / project style guide
  • • Add tests for new functionality
  • • Update documentation as needed

Push your changes to your fork and open a pull request against the main repository. Provide a clear description of your changes and reference any related issues.

  • • Write a clear PR title and description
  • • Link to the related issue
  • • Wait for review and address feedback

We are committed to fostering a welcoming and inclusive community. All contributors are expected to adhere to our Code of Conduct.

  • • Be respectful and inclusive
  • • Provide constructive feedback
  • • Report any violations to the community team
